# 1️⃣ 학습 내용 정리

## 1. 인덱스 개념을 파악하자 

인덱스는 데이터를 빠르게 찾을 수 있도록 도와주는 도구 

1) 클러스터형 인덱스: 기본 키로 지정하면 자동생성됨, 테이블에 1개만 만들 수 있음
2) 보조 인덱스: 고유 키로 지정하면 자동 생성됨, 여러개를 만들수도 있지만 자동정렬되진 않음

### 인덱스의 장단점 
인덱스는 select에서 즉각적 효과를 내는 방법 중  한가지. 즉 적절한 인덱스를 생성하고 인덱스를 사용하는 SQL을 만들면 기존보다 빠른 응답속도를 얻을 수 있음 

인덱스 장점 
- select 문으로 검색하는 속도가 매우 빨라짐
- 그 결과 컴퓨터의 부담이 줄어들어 결국 전체 시스템의 성능이 향상됨

인덱스 단점 
- 인덱스도 공간을 차지해 데이터베이스 안에 추가적인 공간이 필요
- 처음에 인덱스를 만드는데 시간이 오래 걸릴 수 있음

- ### 인덱스의 종류
A. 클러스터형 인덱스 : 영어사전처럼 내용이 이미 알파벳 순서대로 정렬되어 있는 것 
테이블당 1개만 존재 가능
보통 Primary Key에 자동 생성
데이터 페이지 = 리프 노드 (별도 저장 없음)
범위 검색(BETWEEN, >, <)에 매우 빠름
B. 보조 인덱스: 찾아보기가 별도로 있고 찾아보기에서 해당 단어를 찾은 후 옆에 표시된 페이지를 펼쳐야 실제 찾는 내용이 있는 것 
- 고유키로 지정하면 보조 인덱스가 생성됨. 보조 인덱스는 테이블에 여러개 설정 가능 
테이블당 여러 개 생성 가능
추가 저장 공간 필요
조회 시 2단계 탐색: 보조 인덱스 → 클러스터형 인덱스 → 데이터
> **확인문제: 다음은 인덱스 종류와 관련된 설명입니다. 가장 거리가 먼 것을 하나 고르세요.**

보기는 아래와 같습니다.
```
1️⃣ 클러스터형 인덱스는 영어사전과 비슷한 개념입니다.
2️⃣ 보조 인덱스는 일반 책의 찾아보기와 비슷한 개념입니다.
3️⃣ 클러스터형 인덱스는 기본 키를 설정하면 자동 생성됩니다.
4️⃣ 보조 인덱스는 NOT NULL을 설정하면 자동 생성됩니다.
```


## 2. 인덱스의 내부 작동 

### 균형트리의 개념 
균형 트리 구조에서 데이터가 저장되는 공간을 노드라고 한다. 루트 노드는 노드의 가장 상위 노드를 말함. 리프 노드는 제일 마지막에 존재하는 노드를 말함. 데이터가 많다면 3단계나 그 이상이 될 수 있음 

균형트리는 데이터를 검색할 때 뛰어난 성능을 발휘함. 
균형트리는 무조건 루트 페이지부터 검색. 
앞서 데이터를 검색하는 데 균형 트리가 더 효율적이었고, 인덱스는 균형 트리로 구성되어 있음. 

###

> **확인문제: 다음 설명에서 빈칸에 공통으로 들어갈 용어를 쓰시오.**

```
인덱스를 구성하게 되면 데이터의 변경 작업(INSERT, UPDATE, DELETE)시에 성능이 나빠지는 단점이 있습니다.  
특히 INSERT 작업이 일어날 때 더 느리게 입력될 수 있는데요, 이유는 (           ) 이라는 작업이 발생하기 때문입니다.  
(            ) 작업이 일어나면 MySQL이 느려지고 너무 자주 일어나면 성능에 큰 영향을 줍니다.
```

## 2. 실습 문제

1. department 컬럼에 보조 인덱스를 생성하시오.
